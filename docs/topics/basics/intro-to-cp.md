## What is Competitive Programming?

Competitive programming is solving well-defined algorithmic problems under time and memory constraints. Problems are judged automatically — your code either passes all test cases or it doesn't.

## Time Complexity

Time complexity describes how the runtime of an algorithm grows with input size `n`.

| Complexity | Name | Max n (1s) |
|------------|------|------------|
| O(1) | Constant | Any |
| O(log n) | Logarithmic | Any |
| O(n) | Linear | ~10⁸ |
| O(n log n) | Linearithmic | ~10⁷ |
| O(n²) | Quadratic | ~10⁴ |
| O(2ⁿ) | Exponential | ~20 |
